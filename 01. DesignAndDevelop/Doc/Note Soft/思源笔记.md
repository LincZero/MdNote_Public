### 网传的作者人品不好的问题

在这门软件开源之前，我是对这种有收费项、国产、作者人品有问题的 笔记软件，是拒绝的

先来说说作者的混蛋事：（不过近期好像没什么爆料）

> [!quto] 参考：https://www.zhihu.com/question/434530733
>
> 作者干过很多漂亮事，可以参考下面这个存档：
> [使用 Github 账号登录 黑客派 之后， Github 自动 follow](https://link.zhihu.com/?target=https%3A//www.v2ex.com/t/534800)
> [www.v2ex.com/t/534800](https://link.zhihu.com/?target=https%3A//www.v2ex.com/t/534800)
>
> 比较确认并且讨骂的事情有 3 个
>
> 1. 搞了个网站，如果你用 github  登录，github 提供两种登录方式，一种是仅仅告诉第三方你是你的这种授权，一种是作为集成的 app 的方式进去，后者可以拿到很多授权，读写你的仓库代码，关注或者取关某个用户或者项目。通常情况下我们正经的会选择前者，不正经的，或者小白的会选择后者，通常不干净的手法是偷人一个私有邮件地址之类的。但是这位大哥，还要了[授权人](https://www.zhihu.com/search?q=授权人&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra={"sourceType"%3A"answer"%2C"sourceId"%3A2641571976}) 的读写自己项目的权限，然后授权的账号自动关注他个人以及他的某个项目。这个在 github 史上也算是罕见的。
>
>    1. 当时同列的有 github 方式登录，微信，qq 方式登录，但是微信和 QQ 方式“还没维护好”。自己品
>    2. 被举报后，github 官方确认并下了他的那个登录服务，他重新开了个新的，仍然要求获取大量权限，再次给举报没了才老实点
>    3. “道歉“是「给各位带来不便请见谅，稍后会在新手流程中加上一个明显的 star & follow 选项。各位如果觉得我人品有问题，请不要关注。」
>
> 2. 访问他的网站，[你的浏览器会自动跑挖矿脚本](https://link.zhihu.com/?target=https%3A//www.v2ex.com/t/421952)，被指出后回答「你们可能不明白这和挂广告是一样的」，「挖矿并不是什么见不得人的事情，我全部的开源产品都挖了挖矿脚本，并且在 README 里面写明了」
>
>    1. 这儿作者以及他的追随者提出了这样一个论点：「项目首先服务的是项目使用者」，言下之意是不是即便这个损害了访客的利益，也是没问题的？我不这样觉得。不然菠菜，诈骗网站的开发者是不是也可以这样开脱自己？
>
